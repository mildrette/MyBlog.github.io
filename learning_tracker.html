Kotlin — Your Main Language

This is the most important thing you are learning. Kotlin is what your job runs on. Take it seriously but take it slowly. One phase at a time.

1

Phase 1 — The Absolute Basics

Weeks 1 & 2 · Foundation · Learn these before anything else

▼
Week 1
✓
Variables — val and var
val is a permanent box. var is a changeable box. You store things inside them. You already know this one!
⏱ 15 mins Theory
val name = "Mildred" // permanent — cannot change var mood = "tired" // changeable mood = "motivated" // this works!
✓
Data Types — String, Int, Boolean
String = words and sentences. Int = whole numbers. Boolean = true or false. That is it.
⏱ 20 mins Theory
val name: String = "Mildred" val age: Int = 31 val isHappy: Boolean = true
Functions — Writing Your Own
A function is like a recipe. You give it ingredients (parameters) and it gives you back something cooked (return value). Write 3 of your own today.
⏱ 30 mins Build
fun greet(name: String): String { return "Hello $name!" } // call it like this: greet("Mildred") // returns "Hello Mildred!"
If Statements — Making Decisions
IF it is raining THEN take umbrella ELSE go without. Your app makes decisions this way every second.
⏱ 25 mins Build
val isRaining = true if (isRaining) { println("Take umbrella") } else { println("Go without") }
Week 2
Lists — Storing Many Things
Like a shopping list in code. Store many things together. Loop through them. Find things inside them.
⏱ 30 mins Build
val cities = listOf("Bamenda", "Douala", "Yaoundé") cities.forEach { city -> println("I want to visit $city") }
When Expression — Better If Statements
When is like a smarter if. When you have many choices it is cleaner than writing if else if else if else forever.
⏱ 20 mins Theory
val day = "Monday" when (day) { "Monday" -> println("Start of week ") "Friday" -> println("Almost weekend! ") else -> println("Just another day") }
Classes — Describing Things
A class is a blueprint. Like a cookie cutter. You make the shape once then cut many cookies from it. Every Android screen uses classes.
⏱ 40 mins Build
class Person( val name: String, val age: Int ) { fun introduce() { println("Hi I am $name, $age years old") } } val mildred = Person("Mildred", 31) mildred.introduce()
Mini Project — Build a Contact Card
Using everything from Phase 1 — build a simple Contact class with name, phone, and a function that prints their info. No help. Just try.
⏱ 45 mins Project
2

Phase 2 — Jetpack Compose Layouts

Weeks 3 & 4 · Build screens · This is your job now

▼
Week 3
Column — Stack Things Top to Bottom
Column stacks things like a list going downward. Like writing items on a piece of paper top to bottom.
⏱ 20 mins Build
Column { Text("First") Text("Second") Text("Third") } // shows top to bottom
Row — Put Things Side by Side
Row puts things next to each other horizontally. Like buttons sitting in a row.
⏱ 20 mins Build
Row { Text("Left") Text("Right") } // shows side by side
Modifier — Dress Your Components
Modifier is like clothing for your UI. It makes things bigger, smaller, padded, colored, clickable. Everything visual goes through Modifier.
⏱ 30 mins Build
Text( text = "Hello", modifier = Modifier .padding(16.dp) .fillMaxWidth() .background(Color.Yellow) )
Text, Button, Image — The Big Three
These three components make up 80% of every screen you will ever build. Learn them deeply. Practice combining them.
⏱ 40 mins Build
Week 4
remember and mutableStateOf
Your phone forgets everything when it repaints. remember makes it NOT forget. mutableStateOf tells the screen to repaint when the value changes.
⏱ 45 mins Build
var count by remember { mutableStateOf(0) } Button(onClick = { count++ }) { Text("Clicked $count times") }
TextField — Let Users Type
The input box. Every search bar, every message box, every form field is a TextField. Learn to capture what users type.
⏱ 35 mins Build
Mini Project — Profile Screen
Build a complete profile screen with a name, a bio, a button that changes mood, and a text field. No help. Use everything from Phase 2.
⏱ 60 mins Project

Phase 3 — Real App Logic

Weeks 5 & 6 · Make things actually work · ViewModel + Navigation

▼
ViewModel — The Manager
ViewModel is like a manager between your screen and your data. Screen asks manager for data. Manager gets it. Screen displays it. They never talk directly.
⏱ 50 mins Theory + Build
Navigation — Moving Between Screens
How you go from screen A to screen B. Like turning pages in a book. Every app needs this.
⏱ 45 mins Build
LazyColumn — Scrollable Lists
When you have many items — like a list of messages — you use LazyColumn. It only loads what is visible so the app stays fast.
⏱ 40 mins Build
Project — Messages List Screen
Build a screen that looks like a messages list. Profile photo placeholder, name, last message, time. Make it scrollable. This is your Google Messages prep.
⏱ 90 mins Project

Python — For Outreachy & Beyond

Start Python after Phase 2 of Kotlin. Python is easier than Kotlin so it will feel like a breath of fresh air. Many Outreachy projects use Python.

1

Phase 1 — Python Basics

Weeks 1 & 2 · Surprisingly easy after Kotlin

▼
Variables and Data Types
Same concept as Kotlin but simpler. No val or var. No type declarations. Python figures it out for you.
⏱ 20 mins Theory
name = "Mildred" # Python just knows it's a string age = 31 # Python just knows it's a number is_happy = True # True with capital T in Python
Functions in Python
Same idea as Kotlin functions but with def instead of fun. Much simpler syntax.
⏱ 25 mins Build
def greet(name): return f"Hello {name}!" greet("Mildred") # returns "Hello Mildred!"
Lists and Loops
Same as Kotlin lists but even simpler. Loop through them with for. This is the heart of Python scripting.
⏱ 30 mins Build
Reading and Writing Files
Python is amazing at working with files. Open a file, read it, write to it, close it. Very useful for Outreachy projects.
⏱ 35 mins Build
2

Phase 2 — Python for Real Projects

Weeks 3 & 4 · Scripts, APIs, automation

▼
Classes in Python
Same blueprint concept as Kotlin. Build objects with data and functions inside them.
⏱ 40 mins Build
Working with APIs using requests
Python can talk to the internet. Ask a website for data and get it back. This is how real apps talk to servers.
⏱ 50 mins Build
Project — Build a Simple Script
Build a Python script that reads a list of names from a file, says hello to each one, and saves the results to a new file. Real, useful, small.
⏱ 60 mins Project

Phase 3 — Python for Outreachy

Weeks 5 & 6 · Contribute to real projects

▼
